# 项目结构说明

更新时间：`2026-04-10`

## 1. 文档目的

这份文档只回答一个问题：

- 当前仓库里哪些目录是正式活动目录，哪些只是历史归档或参考材料

## 2. 当前正式活动目录

### 2.1 `Seller_Client/`

当前正式卖家客户端目录。

包含：

- 本地 FastAPI Web 壳
- MCP 服务与自然语言编排
- Windows bootstrap 脚本
- 当前卖家接入文档

### 2.2 `Plantform_Backend/`

当前正式平台后端目录。

包含：

- 认证
- seller onboarding
- 节点只读接口
- 文件分发
- 占位 trade/access grant

### 2.3 `Docker_Swarm/`

当前正式 Docker Swarm 基础设施目录。

包含：

- 环境变量
- 运维脚本
- Portainer stack
- `Docker_Swarm_Adapter/`

### 2.4 `wireguard/`

当前正式 WireGuard 基础设施与部署资产目录。

### 2.5 `docs/`

当前活动的协作 runbook 与 CCCC 文档目录。

## 3. 当前辅助但仍保留的目录

### 3.1 `archive/`

保存当前项目自己导出的历史环境快照与迁移前状态。

### 3.2 `_client_backups/`

保存卖家客户端阶段性备份包。

### 3.3 `env_setup_and_install/`

保存环境准备与 Codex 配置模板。

### 3.4 `win_romote/`

保存 Windows 远程 SSH 与辅助脚本说明。

## 4. 当前历史参考目录

### 4.1 `backup_wasted/`

当前主要历史实现与参考材料目录。

里面包括：

- 旧 seller/buyer/client 代码
- 旧 backend 代码
- 旧运维和回滚脚本
- 已废弃设计与备份

它不是当前主开发目录。

## 5. 当前不存在但以前文档里提过的目录

以下目录当前 repo 根下并不存在，不应再被写成“当前主开发目录”：

- `Buyer_Client/`
- `Shared/`

buyer 侧当前只有：

- backend 的 placeholder trade 流程
- `backup_wasted/buyer-client/` 中的历史参考

## 6. 当前建议目录树

```text
/root/Pivot_network
├── Seller_Client/                    # 当前正式卖家客户端
├── Plantform_Backend/                # 当前正式平台后端
├── Docker_Swarm/                     # 当前正式 Swarm 基础设施
│   └── Docker_Swarm_Adapter/         # 当前正式私有基础设施控制面
├── wireguard/                        # 当前正式 WireGuard 资产
├── docs/                             # 当前活动 runbooks / 协作文档
├── archive/                          # 当前项目环境快照
├── _client_backups/                  # 客户端备份包
├── env_setup_and_install/            # 环境安装与配置模板
├── win_romote/                       # Windows 远程辅助说明
├── backup_wasted/                    # 历史实现与废弃参考
├── PROJECT.md
├── 架构说明.md
├── 项目结构说明.md
└── 个人算力交易平台MVP技术方案.md
```

## 7. 当前开发约束

- 新功能优先写入 `Seller_Client/`、`Plantform_Backend/`、`Docker_Swarm/` 对应目录
- 不把 `backup_wasted/` 当主开发目录
- 不再把未来可能存在的 `Buyer_Client/`、`Shared/` 写成“当前已存在”
- 顶层架构说明以当前代码和当前目录现实为准，不以历史规划图为准
